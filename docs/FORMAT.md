# kc-web Deck Builder 格式說明

## 概述

kc-web (制空権シミュレータ) 使用一種特定的 JSON 格式來導入艦隊數據。本插件負責將 poi 的內部數據結構轉換為此格式。

## 完整格式

```javascript
{
  version: 4,           // 格式版本號（固定為 4）
  hqlv: 120,            // 提督等級 (最高練度)
  
  f1: {                 // 第1艦隊
    s1: {               // 第1格 (旗艦)
      i: 1,             // 艦娘 ID
      lv: 80,           // 等級
      is: [1, 2, 3, 4], // 裝備列表 (4個槽位)
      hp: 100,          // HP (可選)
      lu: 10,           // 運 (可選)
      ex: { i: 50 },    // 補強增設 (可選)
      ar: 1,            // 札 (可選)
      sp: 1,            // 海色ribbon(1)/白たすき(2) (可選)
    },
    s2: { /* ... */ },
    s3: { /* ... */ },
    s4: { /* ... */ },
    s5: { /* ... */ },
    s6: { /* ... */ }
  },
  
  f2: { /* 第2艦隊 - 格式同上 */ },
  f3: { /* 第3艦隊 - 格式同上 */ },
  // ... f4 ~ f8
}
```

## 字段說明

### 艦隊層 (f1-f8)

| 字段 | 類型 | 必填 | 說明 |
|------|------|------|------|
| s1-s6 | Object | ✅ | 6個船位，空位可省略 |

### 艦娘層 (s1-s6)

| 字段 | 類型 | 必填 | 說明 |
|------|------|------|------|
| i | Number | ✅ | 艦娘 ID (マスターデータ参照) |
| lv | Number | ✅ | 等級 (1-175) |
| is | Array | ✅ | 裝備列表，長度為槽位數 |

### 裝備格式

```javascript
// 基本裝備
{ i: 1 }           // 僅 ID

// 完整裝備
{ 
  i: 1,             // 裝備 ID
  s: 10,            // 搭載数 (航空系)
  r: 5,             // 改修値 (-3 ~ 最大)
  l: 120            // 熟練度 (0-120)
}
```

### 可選字段

| 字段 | 類型 | 說明 |
|------|------|------|
| hp | Number | HP 值 |
| aa | Number | 對空値 |
| as | Number | 対潜値 |
| lu | Number | 運 (Luck) |
| ex | Object | 補強増設 |
| ar | Number | 札 (1-7海域) |
| sp | Number | 海色ribbon(1)/白たすき(2) |

## poi 數據結構對照

poi 的艦隊數據存儲在 Redux store 中，路徑為 `fleet`：

```javascript
// poi 內部結構 (簡化版)
{
  fleets: [
    {
      ships: [
        {
          id: 1,              // 艦娘 ID
          level: 80,          // 等級
          equipment: [1, 2, 3, 4],  // 裝備列表
          hp: 100,            // HP
          lu: 10,             // 運
          extraEquipment: {   // 補強増設
            id: 50
          }
        }
      ]
    }
  ],
  admiralLevel: 120         // 提督等級
}
```

## 轉換示例

### poi → kc-web

```javascript
// 輸入 (poi)
{
  id: 1,
  level: 80,
  equipment: [1, 2, 3, 4]
}

// 輸出 (kc-web)
{
  i: 1,
  lv: 80,
  is: [1, 2, 3, 4]
}
```

## 注意事項

1. **版本號**: `version` 固定為 4，不要修改
2. **裝備槽位**: 每個艦娘必須有 4 個裝備槽位（即使空槽也需填 0）
3. **補強増設**: 僅部分艦娘支援，需檢查 `extraEquipment` 是否存在
4. **熟練度**: 航空系裝備才有搭載数 (s) 和熟練度 (l)
